# LifePulse v1.2.6 — Release Notes

> UX 体验优化第四批（与 v1.2.5 同主题延续）：把 v1.2.5 验证过的「loading skeleton + 空态 + 错误态可恢复」三态模式，从仅 DailyView / AiDrawer 收口扩展到全站 7 个高频视图 + 抽出 2 个共享组件 + 统一设计 token。
> **Tag**：`v1.2.6` (annotated) on `main` @ `4d3d65c`
> **Date**：2026-07-23
> **对比基线**：`v1.2.5`（`08be097`），`v1.2.5` 后文档 closeout 3 commit + v1.2.6 11 commit = 14 commit 至 `4d3d65c`

---

## 1. Highlights

- **3 处 loading skeleton 新增**（#1-3）：TaskListView（6 行 .task-list-view__skeleton）/ PlanCalendarView（header + 7 周列 + 42 日格 1:1 对齐 CalendarMonth）/ ExpenseView（3 天分组 × 4 列骨架 + 行明细），全部走「`loading=true && list=null`」双条件首次加载骨架屏 + refresh 保留旧数据避免闪烁（v1.2.5 AiDrawer 同款约定）。
- **2 个共享组件**（#4.1）：`TriStateEmpty`（ElEmpty + 命名空间 testId + 默认 slot 留 action 扩展点） + `TriStateError`（ElEmpty + ⚠️ 图标 + 内置「重试」按钮 + `emit('retry')`），各 6 个单测。
- **7 个 view 空态 / 错误态统一迁移**（#4.2-4.7）：DailyView / AiAnalysisView / TaskListView / PlanCalendarView / ExpenseView / DietView + ExpenseList 子组件，全部从散落的内联 `<div>` 与裸 `ElEmpty` 切到 `TriStateEmpty` / `TriStateError`；其中 TaskListView / PlanCalendarView / ExpenseView / DietView 新增「首次加载失败 + list===null」错误态 + 重试入口（v1.2.5 仅 DailyView 有）。
- **三态设计 token 统一**（#5）：新增 `frontend/src/assets/styles/tri-state.css`，定义 `--tri-state-loading-bg` / `--tri-state-loading-radius` 两个 CSS 变量；5 处 view 的 `.skeleton` / `.loading-block` 容器从硬编码颜色 / 圆角切到 token，未来调整主题只改一处。
- **零破坏性变更**：纯前端优化；后端 schema、API、限流、Flyway、CI 闸门、auth / task / plan / expense / diet / settings / ai 既有功能均不受影响。
- **零新增依赖**：沿用 Element Plus 既有 `ElSkeleton` / `ElSkeletonItem` / `ElEmpty` / `ElButton`，0 行 `package.json` 变更。

### 2.6 TaskListView / PlanCalendarView / ExpenseView / DietView 迁移 + 新增错误态（v1.2.6 #4.4-4.7）

| 视图 | 空态 testId | 错误态 testId | 错误态文案 |
|------|------------|---------------|------------|
| TaskListView | `task-list-empty` | `task-list-error` | 1003「无权访问该任务列表」 / 1006「操作过于频繁，请稍后再试」 / 默认「暂时无法获取任务列表，请稍后重试」（按 `store.errorCode` 计算） |
| PlanCalendarView | `day-empty`（day-panel 内）+ `plan-calendar-error`（整页） | `plan-calendar-error` / `plan-calendar-error-retry` | 「暂时无法获取日程数据，请稍后重试」 |
| ExpenseView | ExpenseList 子组件 `expense-list-empty` | `expense-view-error` / `expense-view-error-retry` | 「暂时无法获取消费记录，请稍后重试」 |
| DietView | `diet-view-empty` | `diet-view-error` / `diet-view-error-retry` | 「暂时无法获取饮食记录，请稍后重试」 |

**错误态触发条件**：四个 view 统一为 `store.error && store.list === null`（首次加载失败；refresh 阶段 list 保留旧数据走真实 list，不显示错误态）。

**重试行为**：
- TaskListView → `store.fetchList()`
- PlanCalendarView → `loadMonth()`（= `store.fetchList()`）
- ExpenseView → `Promise.all([refreshList(), refreshSummary()])`（list + summary 双拉）
- DietView → `Promise.all([refreshList(), refreshSummary()])`（list + summary 双拉）

**DietView 优先级**：错误态分支先于空态分支（`v-else-if` 顺序：error → empty → content），避免「loading 失败 → 误显示为空」的反向误导。

**测试新增**：TaskListView.spec.ts +5 / PlanCalendarView.spec.ts +3 / ExpenseView.spec.ts +3 / DietView.spec.ts 待 PR 后统计（合计 +11 用例）。
