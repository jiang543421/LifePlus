## 4. 部署方式

### 4.1 一键 Docker Compose（推荐）

```bash
docker compose up -d --build
```

启动完成后：

| 入口 | 地址 |
|---|---|
| 前端 SPA | `http://localhost` |
| 后端 API | `http://localhost:8080/api/v1` |
| 健康检查 | `http://localhost:8080/actuator/health` |
| MySQL | `localhost:3306`（库 `lifepulse`，用户 `lp` / `lp_dev_only`） |
| Redis | `localhost:6379` |

服务编排（`docker-compose.yml`）：

- `db` — `mysql:8.0`，自带 healthcheck（`mysqladmin ping`）等待就绪
- `redis` — `redis:7-alpine`，healthcheck `redis-cli ping`
- `backend` — 自构 `eclipse-temurin:21-jre` fat jar，等 db+redis 都 healthy 才起
- `frontend` — 自构 `nginx:alpine`，等 backend 起来后才起；SPA 路由 fallthrough 到 `index.html`

### 4.2 拆分开发

```bash
# 终端 1：DB + Redis
docker compose up -d db redis

# 终端 2：后端
cd backend && mvn spring-boot:run   # 用环境变量注入 LP_DB_URL/LP_DB_USER/.../LP_JWT_SECRET

# 终端 3：前端
cd frontend && pnpm install && pnpm dev   # http://localhost:5173，dev 代理 /api → :8080
```

### 4.3 种子账号

MVP1 无 Flyway seed 账号脚本（R-006 待 Phase 5 之后）。本地启动后通过注册接口 `/api/v1/auth/register` 创建首个账号。

### 4.4 环境变量

必需（参考 `docker-compose.yml` + `.env.example`）：

| 变量 | 说明 |
|---|---|
| `LP_DB_URL` | JDBC URL（含 `serverTimezone=Asia/Shanghai`） |
| `LP_DB_USER` / `LP_DB_PASS` | MySQL 凭据 |
| `LP_REDIS_URL` | Redis 连接串 |
| `LP_JWT_SECRET` | **≥ 32 字节**，禁含 `replace-me`（启动 fail-fast） |

### 5.2 E2E 稳定化笔记（CLAUDE.md MEMORY）

- **Element Plus 2.x + Playwright popper-based 组件**：el-select 选项 click 在 Chromium 下偶发不触发 v-model；el-date-picker type=datetime 同源问题。
- **E2E 规避策略**：涉及 `el-select` / `el-date-picker` 取值的场景，改用 `page.evaluate` 直连 pinia store（见 `task-flow.spec.ts: setTaskFilter`、`plan-flow.spec.ts: createPlanViaStore`）。
- **组件级覆盖**：弹层组件的真实 UI 交互由单测覆盖（`EventDialog.spec.ts` 16 用例、`TaskFilters.spec.ts` 6 用例）。
