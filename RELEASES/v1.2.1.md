# LifePulse v1.2.1 — Release Notes

> MVP2 第一阶段：消费（expense）模块落地。
> **Tag**: `v1.2.1` (annotated) on `main` @ `598a89f`（PR #13 merge = `4ab5a87`）。
> **Date**: 2026-07-20。
> **对比基线**: `v1.0.0-mvp`（`0afaf66`）。

---

## 1. Highlights

- **消费模块上线**：单笔 + 预置 5 分类（餐饮 / 购物 / 交通 / 订阅 / 其他）的轻量记账；覆盖「记录 → 列表 → 汇总 → 编辑 → 删除」完整闭环。
- **首页 6 卡升级**：消费卡从 placeholder 升级为 module，直接跳 `/expenses`；剩余 3 占位卡（饮食 / 日报 / AI 分析）保留「即将上线」toast，留待后续 Phase。
- **零破坏性变更**：纯增量；auth / task / plan / settings / home 全部既有功能不受影响；schema、限流、CORS、CSP 沿用 MVP1 约束。
- **TDD 闭环**：T1-T15 全部 Red→Green→Refactor；8 项 AC 全绿，无 TODO/FIXME 残留。

## 2. 新增功能

### 2.1 后端（Spring Boot 3 + MyBatis-Plus + Flyway）

| 模块 | 内容 |
|---|---|
| Flyway V4 | `V4__init_expense.sql` — `t_expense` 表（PK / 2 联合索引 / 2 CHECK 约束 / `@TableLogic`） |
| Entity | `entity/Expense.java` + 软删 + `createTime/updateTime` 自动填充 |
| Enum | `ExpenseCategory` — 5 值（MEAL/SHOPPING/TRANSPORT/SUBSCRIPTION/OTHER）；`IEnum` String 自动转换；与 SQL CHECK 字面值同源 |
| Constants | `ExpenseConstants` — 默认分页大小、最大备注长度等集中常量 |
| DTOs | `CreateExpenseRequest` / `UpdateExpenseRequest` / `ExpenseResponse` / `ExpenseListItem` / `ExpenseFilter` / `ExpensePageResponse` / `ExpenseSummaryResponse` / `CategoryItem` |
| Mapper | 6 SQL：`findByUserAndId` / `listByUser`（filter + 分页） / `countByUser` / `summaryByCategory` / `summaryDeltaMonth` / `summaryDeltaYear` |
| Service | 7 方法：`create / getById / list / update / softDelete / summary / categories`；限流 10/min/userId；越权/不存在 1003/1004；限流触发 1006 |
| Controller | 7 端点：`GET /expenses` / `GET /expenses/{id}` / `POST /expenses` / `PATCH /expenses/{id}` / `DELETE /expenses/{id}` / `GET /expenses/summary` / `GET /expenses/categories` |
| 安全 | `JwtAuthFilter` 全程鉴权 + `UserContext.current()` 过滤；不允许 cross-user 串改 |

### 2.2 前端（Vue 3 + Pinia + Element Plus）

| 模块 | 内容 |
|---|---|

| Store | `stores/expense.ts` — Pinia options API；state：`filter / page / dialogVisible / dialogMode / currentItem`；actions：`fetchList / fetchOne / create / update / remove / fetchSummary / openDialog / closeDialog / setPage` |
| Components | `ExpenseList.vue` + `ExpenseListItem.vue`（分页列表行）；`ExpenseSummaryCard.vue`（HTML 进度条月分类汇总）；`ExpenseDialog.vue`（create/edit 双模态；`el-input-number` 限 2 位小数；空 note trim → null） |

| Home 卡 | `types/home.ts` 中 `expense` 卡 `placeholder: false`；路由跳 `/expenses`；测试卡 spec 已对齐 |
| 错误码映射 | `utils/error.ts` 扩展：1006 → 「操作过于频繁，请稍后再试」；1003 → 「无权操作」；1004 → 「记录不存在」 |

## 5. 升级指南（v1.0.0-mvp → v1.2.1）

### 5.1 兼容性

- ✅ **无破坏性 API 变更**
- ✅ **无 schema 破坏**（纯新增 V4，不修改 V1-V3）
- ✅ **无前端路由冲突**（`/expenses` 全局新增）
- ✅ **无 CI 闸门变更**（沿用 `mvn verify` + `pnpm test` + `pnpm exec playwright test`）

### 5.2 升级步骤

```bash
# 1. 拉取 tag
git fetch origin
git checkout v1.2.1

# 2. 重新构建镜像（自动跑 V4 迁移）
docker compose down
docker compose up -d --build

# 3. 验证 Flyway 迁移
docker compose exec backend \
  java -jar app.jar --spring.flyway.url=jdbc:mysql://db:3306/lifepulse \
  --spring.flyway.user=lp --spring.flyway.password=lp_dev_only \
  --spring.flyway.validate-on-migrate=true \
  --spring.flyway.locations=classpath:db/migration
# 期望：Successfully applied 4 migrations to schema ... (V1 ~ V4)
```

### 5.3 回滚步骤

```bash
# 1. 停止服务
docker compose down

# 2. 回退代码 + 回退迁移（V4 加 IF EXISTS 兼容性脚本）
git checkout v1.0.0-mvp
docker compose up -d --build

# 3. 删除 expense 表（可选）
docker compose exec db mysql -ulp -plp_dev_only lifepulse \
  -e "DROP TABLE IF EXISTS t_expense;"
```

### 5.4 数据迁移提示

- V4 是纯新增表，旧数据无需迁移
- 升级前**无需**手工操作
- 升级后**新增**的能力：消费记录、首页消费卡入口；既有数据不受影响
