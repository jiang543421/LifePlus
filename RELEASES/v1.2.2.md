# LifePulse v1.2.2 — Release Notes

> MVP2 第二阶段：饮食（diet）模块落地 + R-006 / Bug #1 收口。
> **Tag**: `v1.2.2` (annotated) on `main` @ TBD-PR-merge-commit。
> **Date**: 2026-07-21。
> **对比基线**: `v1.2.1`（`598a89f` / `4ab5a87` PR #13 merge）。

---

## 1. Highlights

- **饮食模块上线**：单条记录 + 4 类餐饮（早/午/晚/加餐）+ 热量统计 + 「常吃」聚合的轻量记录；覆盖「记录 → 列表 → 汇总 → 常吃 → 编辑 → 删除」完整闭环。
- **首页 6 卡升级**：饮食卡从 placeholder 升级为 module，直接跳 `/diets`；剩余 2 占位卡（日报 / AI 分析）保留「即将上线」toast，留待后续 Phase。
- **R-006 收口**：`dev` profile 不再追加 `classpath:db/seed`；`UserIT` 删 `@ActiveProfiles("dev")` 与 `flywayHistory_includesV5`；种子账号改走 `@BeforeEach` 幂等 INSERT（与 V5 Flyway 解耦）。
- **Bug #1 收口**：`AuthServiceIT` 的 `expiresIn` 断言对齐 `JwtProperties.accessTtl`（PT15M），不再硬编码 `AuthConstants.ACCESS_TTL`（PT1H）。
- **零破坏性变更**：纯增量 + 收口；auth / task / plan / expense / settings / home 全部既有功能不受影响；schema、限流、CORS、CSP 沿用 v1.2.1 约束。
- **TDD 闭环**：T1-T22 全部 Red→Green→Refactor；无 TODO/FIXME 残留。

## 2. 新增功能

### 2.1 后端（Spring Boot 3 + MyBatis-Plus + Flyway）

| 模块 | 内容 |
|---|---|
| Flyway V6 | `V6__init_diet.sql` — `t_diet` 表（PK / 2 联合索引 / 4 CHECK 约束 / `@TableLogic`） |
| Entity | `entity/Diet.java` + 软删 + `createTime/updateTime` 自动填充 |
| Enum | `MealType` — 4 值（BREAKFAST/LUNCH/DINNER/SNACK）；`IEnum` String 自动转换；与 SQL CHECK 字面值同源 |
| Constants | `DietConstants` — 默认分页大小、最大备注长度、最大 name 长度等集中常量 |
| DTOs | `CreateDietRequest` / `UpdateDietRequest` / `DietResponse` / `DietListItem` / `DietFilter` / `DietPageResponse` / `DietSummaryResponse` / `MealTypeSummaryItem` |
| Mapper | 6 SQL：`findByUserAndId` / `listByUser`（filter + 分页）/ `countByUser` / `summaryByMealType` / `summaryDeltaDay` / `frequentByUser` |
| Service | 7 方法：`create / getById / list / update / softDelete / summary / frequent`；限流 10/min/userId；越权/不存在 1003/1004；限流触发 1006 |
| Controller | 7 端点：`GET /diets` / `GET /diets/{id}` / `POST /diets` / `PATCH /diets/{id}` / `DELETE /diets/{id}` / `GET /diets/summary` / `GET /diets/frequent` |
| 安全 | `JwtAuthFilter` 全程鉴权 + `UserContext.current()` 过滤；不允许 cross-user 串改 |

## 5. 升级指南（v1.2.1 → v1.2.2）

### 5.1 兼容性

- ✅ **无破坏性 API 变更**
- ✅ **无 schema 破坏**（纯新增 V6，不修改 V1-V5）
- ✅ **无前端路由冲突**（`/diets` 全局新增）
- ✅ **无 CI 闸门变更**（沿用 `mvn verify` + `pnpm test` + `pnpm exec playwright test`）

### 5.2 升级步骤

```bash
# 1. 拉取 tag
git fetch origin
git checkout v1.2.2

# 2. 重新构建镜像（自动跑 V6 迁移）
docker compose down
docker compose up -d --build

# 3. 验证 Flyway 迁移
docker compose exec backend \
  java -jar app.jar --spring.flyway.url=jdbc:mysql://db:3306/lifepulse \
  --spring.flyway.user=lp --spring.flyway.password=lp_dev_only \
  --spring.flyway.validate-on-migrate=true \
  --spring.flyway.locations=classpath:db/migration
# 期望：Successfully applied 6 migrations to schema ... (V1 ~ V6)
```

### 5.3 回滚步骤

```bash
# 1. 停止服务
docker compose down

# 2. 回退代码 + 回退迁移（V6 加 IF EXISTS 兼容性脚本）
git checkout v1.2.1
docker compose up -d --build

# 3. 删除 diet 表（可选）
docker compose exec db mysql -ulp -plp_dev_only lifepulse \
  -e "DROP TABLE IF EXISTS t_diet;"
```

### 5.4 数据迁移提示

- V6 是纯新增表，旧数据无需迁移
- 升级前**无需**手工操作
- 升级后**新增**的能力：饮食记录、首页饮食卡入口；既有数据不受影响
- **R-006 收口影响**：种子 demo 账号（`demo@lifepulse.test` / `alice@lifepulse.test`）**不再由 Flyway V5 自动注入**。生产环境如需演示数据，改为：
  ```bash
  # 通过注册接口创建
  curl -X POST http://localhost:8080/api/v1/auth/register \
    -H "Content-Type: application/json" \
    -d '{"email":"demo@lifepulse.test","password":"Demo123!","nickname":"demo"}'
  ```
