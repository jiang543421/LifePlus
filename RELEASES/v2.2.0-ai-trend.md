# LifePulse v2.2.0-ai-trend — Release Notes

> MVP2 第六阶段：**AI 趋势图 v2.2** —— 在 v2.1 的"今日快照 + 独立分析页"基础上叠加 7/14/30 天趋势图，4 槽 metrics（task / plan / expense / diet），独立分析页 `/ai-analysis` 新增「趋势」段。
> **Tag**: `v2.2.0-ai-trend`（annotated）on `main` @ `feat/ai-trend-chart` squash merge。
> **Date**: 2026-07-23。
> **对比基线**: `v2.1.0-ai`（`4d81aa5`）。

---

## 1. Highlights

- **趋势图段落地**：`/ai-analysis` 在 chips 段后追加「趋势」段，4 槽 2×2 网格（task 任务完成率 / plan 日程事件 / expense 消费金额 / diet 永久占位），桌面 2 列、移动 1 列堆叠。
- **3 窗口切换**：ElRadioButton 7 天 / 14 天（默认）/ 30 天，切换立刻拉数据；URL 同步 `?window=N`（`router.replace` 不增加历史），便于分享。
- **窗口分桶缓存**：前端 `useAiTrendStore` 按 window 维度分 3 槽（7/14/30），切换不互相覆盖，旧槽位保留避免骨架屏闪烁。
- **后端只读 5 张既有表**：CLAUDE.md §11.1 硬约束 1-2 — `t_task` / `t_plan` / `t_expense` / `t_daily_report` / `t_diet` 字段语义零变，**0 个新 Flyway 迁移**。
- **零新依赖**：手写 inline SVG sparkline（viewBox 100x48 + polyline + endpoint dot + ElTooltip overlay），复用既有 `ElRadioGroup` / `ElRadioButton` / `ElSkeleton` / v1.2.6 `TriStateEmpty` / `TriStateError`。
- **Redis 6h 缓存 + 30/min/user 限流**：与 `/today` / `/analysis` 同档（CLAUDE.md §11.4）；缓存键 `lp:ai:trend:<userId>:<windowDays>`，限流键 `lp:rl:ai:trend:<userId>`。
- **14 个后端单测 + 7 个前端 TrendPanel 测试**：每个测试独立 commit（CLAUDE.md §5.3 节奏），C2 8 个方法 13 个测试用例（参数化）、C9 7 个 vitest（覆盖渲染 / 拉数据 / skeleton / error+retry / 窗口切换 / URL 同步）。
- **功能优先节奏**：C4 / C6 / C8 / C10（IT / 单测补丁）显式后置到趋势图功能稳定后补（与 CLAUDE.md §11.7 v2.1 阈值相比，仅覆盖已落地路径）。

## 2. 新增功能

### 2.1 后端（Spring Boot 3.3.5 + MyBatis-Plus 3.5.6）

| 模块 | 内容 |
|---|---|
| 包结构（演进）| `com.lifepulse.ai` — `AiConstants` 追加 7 个 `TREND_*` 常量；`service/AiTrendService` 新增 `getTrend(userId, windowDays)`；`web/AiTrendController` 新建 |
| 端点 | `GET /api/v1/ai/insight/trend?window=7|14|30`，window 默认 14；非法 window → 1001 VALIDATION |
| 鉴权 | `@AuthenticationPrincipal Long userId` 解析（CLAUDE.md §11.1 硬约束 6：不接受 userId 形参） |
| 限流 | 30/min/user，`lp:rl:ai:trend:<userId>`；超限 → 1006 LOGIN_RATE_LIMIT（复用既有 ErrorCode） |
| 缓存 | `lp:ai:trend:<userId>:<windowDays>` TTL 6h；Redis 不可达 fail-open（直接走计算） |
| 聚合 | 按 `DailyReportPayload` 4 槽（task / plan / expense / diet）按日填充；diet 槽位永久 `enabled=false` 输出空 points（CLAUDE.md §1 NOT-DO） |
| 性能预算 | 7 天 ≤ 250ms / 14 天 ≤ 500ms / 30 天 ≤ 800ms（mapper 调用 28/56/120 次） |
| 错误码 | 无新增；复用既有 1001 / 1006 |

## 5. 升级指南（v2.1.0-ai → v2.2.0-ai-trend）

### 5.1 兼容性

- ✅ **无破坏性 API 变更**（v2.1 的 4 个 AI 端点 100% 兼容）
- ✅ **无 schema 变更**（CLAUDE.md §11.1 硬约束）
- ✅ **0 个新 Flyway 迁移**
- ✅ **无前端路由冲突**（`/ai/insight/trend` 与既有 `/today` / `/analysis` 并列）
- ✅ **无 CI 闸门硬性变更**（沿用 `mvn verify` + `pnpm vitest`）
- ⚠️ **无新增环境变量**（沿用 v2.1 配置）

### 5.2 升级步骤

```bash
# 1. 拉取 tag
git fetch origin
git checkout v2.2.0-ai-trend

# 2. 启动主栈（沿用 v2.1 .env）
docker compose up -d --build

# 3. 验证后端健康
curl -fsS http://localhost:8080/actuator/health
# 期望：{"status":"UP",...}

# 4. 验证趋势端点
curl -fsS -H "Authorization: Bearer ${ACCESS_TOKEN}" \
  'http://localhost:8080/api/v1/ai/insight/trend?window=14' | jq .
# 期望：code=0；data.series 含 task / plan / expense / diet 4 槽
#   diet.points 为空数组；其它 3 槽按日期填充

# 5. 浏览器验证
# 登录 → /ai-analysis → 滚动到「趋势」段 → 看到 4 槽 2×2 网格
# 切换 7 天 / 14 天 / 30 天 → 图表刷新 + URL ?window= 同步
```

### 5.3 回滚步骤

```bash
# 1. 停止服务
docker compose down

# 2. 回退代码
git checkout v2.1.0-ai
docker compose up -d --build

# 3. 清理 v2.2 缓存（可选，无害，不影响 v2.1）
docker compose exec redis redis-cli \
  --scan --pattern 'lp:ai:trend:*' | xargs -r redis-cli DEL
docker compose exec redis redis-cli \
  --scan --pattern 'lp:rl:ai:trend:*' | xargs -r redis-cli DEL
```

### 5.4 数据迁移提示

- **无 schema 变更**；旧数据无需迁移
- 升级前**无需**手工操作
- 升级后**新增**的能力：
  1. `/ai-analysis` 趋势段（4 槽 7/14/30 天）
  2. URL `?window=N` 同步窗口选择
  3. 切换窗口不互相覆盖（3 槽独立缓存）
- **首次激活体验**：diet 槽位永久显示空态（"饮食（永久占位）"），与 spec 一致；其它 3 槽按用户历史数据填充。
