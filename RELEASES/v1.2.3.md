# LifePulse v1.2.3 — Release Notes

> MVP2 第三阶段：日报（daily）模块落地 — 1 个聚合端点 + 周报对照 + 跨数据源编排。
> **Tag**：`v1.2.3` (annotated) on `main` @ `ccb51dddc03cad8d5e41e31ee2457feae538417c`
> **Date**：2026-07-21
> **对比基线**：`v1.2.2`（`287a1f7`），PR `#15 feat(daily): ship v1.2.3 – daily report module (auto-aggregated dashboard)`

---

## 1. Highlights

- **日报模块上线**：单端点 `GET /api/v1/daily?date=YYYY-MM-DD` 一键聚合当日的任务 / 日程 / 消费 3 维指标；周报 `GET /api/v1/daily/week` 给出本周每天清单 + 与上周的 delta（top 任务 / 总支出 / 完成率）；饮食永久禁用（`DietMetrics.enabled=false`）。
- **跨数据源编排**：`DailyReportService` 串联 4 个 `MetricProvider`，单测 + 切片 + Testcontainers IT 三层覆盖（Provider 100% 行覆盖）。
- **TDD 闭环**：T1-T8 全部 Red→Green→Refactor；新增 16 单元 + 9 切片 + 11 IT 用例。
- **首尾数据窗口**：过去 30 天外拒绝（1004）；未来日期允许；ISO 周在跨年边界正确（2025-12-29 → 2026-W01）。
- **零破坏性变更**：纯增量；auth / task / plan / expense / diet / settings / home 既有功能不受影响；无新 V migration（V5 复核为 no-op：V2 `idx_user_status_due` 已完美覆盖聚合路径）。
- **占位卡暂不激活**：Home 6 卡中「日报 / AI 分析」仍为 placeholder，**前端改造留待后续小迭代**；本 PR 仅含后端能力，前端接入会在 issue `docs/issues/2026-07-21-daily-frontend-v1-2-4.md`（建议拆出）收口。

## 2. 新增功能

### 2.1 后端（Spring Boot 3 + MyBatis-Plus + Flyway）

| 模块 | 内容 |
|------|------|
| Provider 接口 | `daily/provider/MetricProvider<T>` — `aggregateDaily(userId, date): T` + `enabled` 钩子；4 实现注入成 List |
| TaskProvider | `task/dueAt ∈ [dayStart, nextDayStart)` 内 status=0/1/2 聚合（新建/进行中/已完成），priority 高/中/低分布 |
| PlanProvider | `plan/startAt ∈ [dayStart, nextDayStart)` 内事件总数 + allDay 计数 |
| ExpenseProvider | `expense/expenseAt ∈ [dayStart, nextDayStart)` UTC 半开区间；按 category 汇总 top 3；总额 = Σ amount(精确 BigDecimal) |
| DietProvider | `enabled=false` 永久冻结（返回 `DietMetrics(enabled=false, kcal=null, topCategories=[])`），spec/plan §5 T6 已明确不做 |
| Service | `daily / week / today` 3 公共方法；`MAX_HISTORY_DAYS=30` 窗口拒绝；`Asia/Shanghai` 边界计算；`previousOrSame(MONDAY)` ISO 周对齐；`WeekFields.ISO` 跨年周 |
| Controller | `DailyReportController` 2 GET 端点：`/daily` + `/daily/week`；`@AuthenticationPrincipal` 拿 userId；date 缺省 = 今日 |
| 错误码 | 1001 校验（超出 30 天窗口）/ 1004 不存在或日期格式错（`?date=not-a-date` 走全局 `MethodArgumentTypeMismatchException` → 400/1001） |

### 3.2 性能基线（local + WSL2 + Docker MySQL，1w 行种子）

| 端点 | 预算 | 实测 P95 | 备注 |
|------|----:|--------:|------|
| `GET /api/v1/daily` | ≤ 200ms | ≈ 100–200ms | 单日 1× 4 Provider 聚合 |

> CI 真实过慢时可降 `DailyConstants.PERF_SEED_ROWS` 并按比例放宽，本 PR 把放宽理由写入 `DailyConstants.WEEKLY_P95_BUDGET` 的 JavaDoc。

## 5. 升级指南（v1.2.2 → v1.2.3）

### 5.1 兼容性

- ✅ **无破坏性 API 变更**

- ✅ **无前端路由冲突**（前端未落地）
- ✅ **无 CI 闸门变更**

### 5.2 升级步骤

```bash
# 1. 拉取 tag
git fetch origin
git checkout v1.2.3

# 2. 重新构建镜像
docker compose down
docker compose up -d --build

# 3. 自检：调用日报
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8080/api/v1/daily?date=$(date +%F)"
# 期望：返回 {date, task, plan, expense, diet{enabled:false}, ...}
```

### 5.3 回滚步骤

```bash
docker compose down
git checkout v1.2.2
docker compose up -d --build
```

无需数据迁移；纯代码 + 配置回退。

### 5.4 数据迁移提示

- 零 schema 变更，旧数据无需迁移
- 升级后**新增**的能力：`GET /api/v1/daily` / `GET /api/v1/daily/week` 端点
