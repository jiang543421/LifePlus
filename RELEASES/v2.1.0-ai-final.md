## 3. 关键功能交付清单

### 3.1 LLM 双客户端（DeepSeek + Ollama）

| 客户端 | 协议 | 关键点 |
|---|---|---|
| `DeepSeekClient` | OpenAI 兼容 `/v1/chat/completions` | 5s 超时；Bearer auth；非流式；解析 `choices[0].message.content` → JSON |
| `OllamaClient` | 本地 `/api/chat` | 不需要 API key；`format: "json"` 提示；与 DeepSeek 输出 schema 同构 |
| 切换点 | `lp.ai.llm.provider=deepseek|ollama` | 启动期通过环境变量；运行期不切换（CLAUDE.md §11.8 反模式清单）|

### 3.2 3 层降级链路（hard rule · CLAUDE.md §11.3）

```
L1 LLM 成功  → source="llm"，渲染完整 payload
L1 LLM 失败  → catch Llm*Exception → L2 模板 → source="template"
L2 模板失败  → throw BusinessException(1501) → HTTP 503
```

- **L1 失败不重试**（5s 超时已吸收抖动；429 重试只会更糟）
- **L2 失败不返回部分数据**（1501 语义是完全不可用）
- **用户感知最小化**：source 标签 + 缺少 advice/highlight 段，**无错误弹窗**
- **触发场景 A 类**（LLM 失败）：5xx / 429 / 4xx / 超时 / 解析失败 / 敏感词
- **触发场景 B 类**（绕开 L1）：4 chip 全 `none()` / `enabled=false` / 配额超限 / 熔断中

### 3.3 配额控制（CLAUDE.md §11.5）

- **每日**：50 调用 / 用户 / 天（`lp.ai.llm.daily-quota`，范围 1–1000）
- **Redis 键**：`lp:ai:quota:<userId>:<yyyymmdd>`（String），TTL = 25h（首次 INCR 时 EXPIRE）
- **触发**：INCR 后超阈值 → `BusinessException(1510)` → Service catch → L2 模板
- **降级**：Redis 不可用时 **fail-open**（放行；用户体验优先于精度）

### 3.4 熔断器（CLAUDE.md §11.5）

- **阈值**：10 失败 / 5min（`lp.ai.llm.circuit-breaker.failures`，范围 1–100 / 1–60min）
- **恢复**：30min（`lp.ai.llm.circuit-breaker.recovery`，范围 1–1440min）
- **Redis 键**：`lp:ai:circuit:state`（OPEN/CLOSED） + `:openedAt`（永久）+ `:failures`（Sorted Set，5min 滑动）
- **Ollama 模式特殊**：`circuit-breaker.enabled=false` 自动禁用（本地进程死掉 ≠ 远程故障）
- **降级**：Redis 不可用时 **fail-closed**（不熔断）
- **触发**：进入 OPEN → `BusinessException(1511)` → Service catch → L2 模板

### 3.5 前端独立分析页 `/ai-analysis`

- **路径**：`/ai-analysis`
- **鉴权**：全局 `beforeEach` 守门；未登录跳 `/login?redirect=/ai-analysis`
- **4 个 section**（v2.1 新增）：
  - `data-testid="ai-analysis-headline"` — 一句话总结
  - `data-testid="ai-analysis-advice"` — 操作建议
  - `data-testid="ai-analysis-highlight"` — 关键亮点
  - `data-testid="ai-analysis-chips"` + `ai-analysis-chip` — 4 chip（Chip keys 来自 `Chips` 枚举：NONE/POSITIVE/NEGATIVE 等，spec 中 3 个激活 key）
- **降级感知**：`data-testid="ai-analysis-degraded-hint"`（`source="template"` 时显示 `ElAlert` 提示）
- **空态**：`data-testid="ai-analysis-empty"`（`insight=null` 时 `ElEmpty`）
- **加载**：`ElSkeleton` 占位
- **刷新按钮**：`data-testid="ai-analysis-refresh"`（POST `refresh`，限流 3/min/user）

### 3.6 source 标签

- **后端字段**：`AiInsightResponse.source: "llm" | "template"`
- **前端显示**：
  - `"llm"` → `AI 智能`（紫色 `ElTag`，首页 AI 卡 + 独立分析页头部）
  - `"template"` → `模板`（灰色 `ElTag`）
- **位置**：首页 ModuleCard 的副标题区 + `AiAnalysisView` 页面 `data-testid="ai-analysis-source-tag"`
- **设计意图**：用户知道当前看到的内容是 LLM 实时生成还是模板兜底（来源透明，无营销化）

### 3.7 6h Redis 缓存 TTL

- **键**：`lp:ai:insight:<userId>`（String/JSON）
- **TTL**：**6 小时**（v2.0 是 30 分钟；v2.1 拉长以摊薄 LLM 成本）
- **范围**：60s–24h（可配置 `lp.ai.llm.cache-ttl`）
- **降级**：Redis 不可用 → **MISS**（走计算路径，无缓存命中）

### 3.8 抽屉"查看完整分析"跳转

- **触发**：首页 AI 卡 → 点 `el-drawer` 打开 `AiDrawer`
- **入口**：`AiDrawer` 底部新增按钮（v2.1） → 点击跳 `/ai-analysis`
- **数据共享**：`useAiInsightStore` 已是单例，路由跳转后 store 内的 `insight` 直接渲染
- **freshness-aware**：`AiAnalysisView.onMounted` 调 `store.loadAnalysis()`，若 `isFresh && insight` 跳过网络请求（避免冗余 GET）

## 5. 与 v2.0.0-ai 的差异

### 5.1 新增能力

| 维度 | v2.0.0-ai | v2.1.0-ai |
|---|---|---|
| LLM 调用 | 0（纯模板） | **1 路径**（DeepSeek + Ollama 二选一） |
| 降级链路 | L1 provider 失败 → L2 模板 → 1501 | **L1 provider 失败 → L1 LLM 失败 → L2 模板 → 1501** |
| 缓存 TTL | 30min | **6h**（拉长 12× 摊薄成本）|
| Rate limit `POST /refresh` | 6/min/user | **3/min/user**（收严，避免用户在 LLM 抖动时狂点）|
| Frontend AI 入口 | 抽屉内详情 | **抽屉 + 独立分析页 `/ai-analysis`**（4 section 可独立访问）|
| source 标签 | 无 | **`"AI 智能"` / `"模板"` 标签**（来源透明）|
| 每日配额 | 无 | **50 次/用户/天**（v2.1 新增；防滥用 + 成本控制）|
| 熔断器 | 无 | **10 fails / 5min → OPEN 30min**（v2.1 新增）|
| 用户级 prompt 模板 | 无 | **`llm-prompt.properties` 注入占位符**（v2.1 新增；支持运行时调 prompt 不重发布）|

### 5.2 架构变化

| 层 | v2.0.0-ai | v2.1.0-ai |
|---|---|---|
| 调用链 | Controller → `AiInsightService` → Provider → Mapper | **Controller → `AiInsightService` → `LlmInsightGenerator` → `LlmClient` → Mapper** |
| 新组件 | — | `LlmInsightGenerator` / `LlmPromptBuilder` / `LlmJsonParser` / `LlmQuotaGuard` / `LlmCircuitBreaker` |
| 新错误码 | 11 个 | **15 个**（+1510 quota / +1511 circuit / +1512 invalid / +1513 unavailable — 全部由 Service catch → L2，几乎不直接给用户）|
| API 端点 | 34 个 | **35 个**（+1 `/api/v1/ai/insight/analysis`）|
| Redis 命名空间 | `ai:insight:*` / `lp:rl:*` / `daily:*` | ... + **`lp:ai:insight:*` / `lp:ai:quota:*` / `lp:ai:circuit:*`** |
| 模块划分 | `auth / task / plan / expense / diet / daily / ai` | ... + **`ai.llm.*`**（独立 LLM 子包）|
| Flyway 迁移 | V1–V6 | **V1–V6 不变**（CLAUDE.md §11.1 硬约束：v2.1 零迁移） |

### 5.3 性能与成本变化

| 项 | v2.0.0-ai | v2.1.0-ai | 评估 |
|---|---|---|---|
| 用户等待 | <200ms（纯模板）| 1–5s（LLM 阻塞）| 视觉：抽页 skeleton 占位（`ElSkeleton`）|
