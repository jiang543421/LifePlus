# LifePulse v2.0.0-ai — Release Notes

> MVP2 第四阶段：**AI 分析 v2.0** 模块落地（智能卡 + 详情抽屉）。
> **Tag**: `v2.0.0-ai` (annotated) on `main` @ `946995b`（PR #16 squash merge）。
> **Date**: 2026-07-22。
> **对比基线**: `v1.2.3`（`ccb51dd` PR #15 merge）。

---

## 1. Highlights

- **AI 分析智能卡上线**：首页 6 卡中"AI 分析"从占位升级为 module，点击打开 `AiDrawer` 抽屉，显示 1-2 句中文 headline + 3 个关键指标 chip + 手动 `🔄` 刷新按钮。
- **5 个 Provider 聚合**：Task（今日完成率） / Plan（今日日程数） / Expense（本周消费） / Diet（今日摄入） / Daily（连续天数，stub 默认禁用） — 任一模块未上线 / 抛异常不阻塞整体，单独降级。
- **30 分钟 Redis 缓存** + 手动 `/refresh`：GET 命中不延长 TTL；POST `/refresh` 清缓存并立即重算；模板采用 JDK `String.format` + `ai-templates.properties`，无 LLM 调用、无新数据库表。
- **端到端鉴权 + 限流**：JWT access token 强制；GET 60/min/user、POST 6/min/user（与既有 `RateLimiter` 一致，复用 `lp:rl:ai:insight:` 前缀）。
- **降级码 1501（`AI_DEGRADED`）**：所有 enabled provider 同时失败时抛 `BusinessException(1501)` → 503，前端 Toast「AI 洞察数据暂时不可用」；限流命中复用 1006。
- **零破坏性变更**：纯增量 + 占位卡升级；task / plan / expense / diet / daily / auth 全部既有功能不受影响；schema、限流（既有模块）、CORS、CSP 沿用 v1.2.3。
- **TDD 闭环**：T1-T11 全部 Red→Green→Refactor；无 TODO/FIXME 残留。

## 2. 新增功能

### 2.1 后端（Spring Boot 3 + MyBatis-Plus + Flyway）

| 模块 | 内容 |
|---|---|
| 包结构 | `com.lifepulse.ai.{constants,properties,web.dto,service,provider,model}` |
| Constants | `AiConstants` — 缓存键前缀 `ai:insight:`、限流键前缀 `lp:rl:ai:insight:`、chip 数 `MAX_CHIPS=3`、TTL 常量等 |
| Properties | `AiInsightProperties` (`@ConfigurationProperties("lp.ai")`) — 5 个 provider toggles + cache ttl + freshness 钳位 |
| DTOs | `AiInsightResponse`（headline + chips + generatedAt + freshnessSeconds）；`AiChipDto`（key / label / value / unit / trend / deltaText） |
| 模型 | `AiInsightPayload`（内部 record）；`MetricValue`（value + delta + trend）；`Trend` 枚举 `UP/DOWN/FLAT/NONE` |
| Provider 接口 | `AiInsightProvider` — `key() / isEnabled(userId) / collect(userId, ctx)` |
| Provider 实现 | `TaskAiProvider` / `PlanAiProvider` / `ExpenseAiProvider` / `DietAiProvider` / `DailyAiProvider`（stub，默认 `isEnabled=false`） |
| Service | `AiInsightService` — 编排 + Redis 缓存 + 模板渲染 + 降级捕获；任一 provider 抛异常 `log.warn` 后跳过，全失败 → `BusinessException(1501)` |
| 模板引擎 | `AiTemplateEngine` — `ai-templates.properties` + JDK `String.format`，UTF-8 加载；11+ 模板键（headline.full / headline.taskOnly / headline.expenseOnly / headline.empty / chip.*.up / down / flat / none 等） |
| Controller | `AiInsightController` — `GET /api/v1/ai/insight/today`（access token + 60/min/user）、`POST /api/v1/ai/insight/refresh`（access token + 6/min/user）；现算 `freshnessSeconds = max(0, Duration.between(generatedAt, now()).getSeconds())` |
| 安全 | `JwtAuthFilter` 全程鉴权 + `UserContext.current()` 过滤；不允许 cross-user 串访 |

## 5. 升级指南（v1.2.3 → v2.0.0-ai）

### 5.1 兼容性

- ✅ **无破坏性 API 变更**
- ✅ **无 schema 变更**（无新 migration；AI 模块只读 5 个既有表）
- ✅ **无前端路由冲突**（`/api/v1/ai/insight/*` 是 API 路径，非前端路由）
- ✅ **无 CI 闸门变更**（沿用 `mvn verify` + `pnpm test` + `pnpm exec playwright test`）
- ⚠️ **Redis 命名空间新增** `ai:insight:*` 与 `lp:rl:ai:insight:*` — 旧 Redis 实例可保留（不同前缀，无冲突）

### 5.2 升级步骤

```bash
# 1. 拉取 tag
git fetch origin
git checkout v2.0.0-ai

# 2. 重新构建镜像（无 Flyway 迁移）
docker compose down
docker compose up -d --build

# 3. 验证后端健康
curl -fsS http://localhost:8080/actuator/health
# 期望：{"status":"UP",...}

# 4. 验证 AI 端点（需先登录获取 access token）
curl -fsS -H "Authorization: Bearer ${ACCESS_TOKEN}" \
  http://localhost:8080/api/v1/ai/insight/today | jq .
# 期望：code=0；data.headline 非空；data.chips 长度 1-3
```

### 5.3 回滚步骤

```bash
# 1. 停止服务
docker compose down

# 2. 回退代码（无 schema 变更，无需回退迁移）
git checkout v1.2.3
docker compose up -d --build

# 3. 清理 AI 缓存（可选）
docker compose exec redis redis-cli \
  --scan --pattern 'ai:insight:*' | xargs -r redis-cli DEL
docker compose exec redis redis-cli \
  --scan --pattern 'lp:rl:ai:insight:*' | xargs -r redis-cli DEL
```

### 5.4 数据迁移提示

- 无 schema 变更；旧数据无需迁移
- 升级前**无需**手工操作
- 升级后**新增**的能力：首页 AI 智能卡 + 抽屉 + 5 Provider 聚合洞察；既有数据完全保留
- **首次激活体验**：历史数据自动进入 Provider 聚合（Task/Plan/Expense/Diet/Daily stub）；若某 Provider 数据为空 → chip `value="—"` / `trend=NONE`；无需引导数据导入
