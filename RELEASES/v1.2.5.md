# LifePulse v1.2.5 — Release Notes

> UX 体验优化第三批：3 处用户感知度最高的"白屏/无引导"问题收口 — 抽屉首次加载骨架屏 + AI 角标语义提示 + 日报错误态可恢复入口。
> **Tag**：`v1.2.5` (annotated) on `main` @ `08be097`
> **Date**：2026-07-23
> **对比基线**：`v2.1.0-ai`（`08be097` 即在其上），3 commit 直推 `main`

---

## 1. Highlights

- **AiDrawer loading skeleton**：用户首次点击首页 AI 卡打开抽屉时，从空白等待改为 1 个 headline + 3 个 chip 的骨架屏占位，消除"打开了却看不到东西"的卡顿感；refresh 时仍保留旧 insight（按钮自身 loading 态覆盖，不重复渲染 skeleton）。
- **AI 角标 hover tooltip**：首页 AI 卡右上角 source 角标（"AI 智能" / "模板"）新增 `ElTooltip`（placement=top, effect=light），hover 时说明 LLM 智能生成 vs 模板降级的语义差异。
- **DailyView 错误态友好化**：日报接口失败时，从「`ElEmpty description="暂无日报数据，请稍后重试"`」无操作引导升级为图标 + 文案「暂时无法获取日报数据，请稍后重试」 + 「重试」按钮（自动按当前 URL 参数重新调 `daily` + `week`）。
- **零破坏性变更**：纯前端优化；后端 schema、API、限流、Flyway、CI 闸门、auth / task / plan / expense / diet / settings / ai 既有功能均不受影响。
- **零新增依赖**：沿用 Element Plus 既有 `ElSkeleton` / `ElSkeletonItem` / `ElTooltip` / `ElEmpty` 组件，0 行 `package.json` 变更。

## 2. 新增功能（前端 UX）

### 2.1 AiDrawer loading skeleton（v1.2.5 #1）

| 维度 | 详情 |
|------|------|
| 文件 | `frontend/src/components/AiDrawer.vue`（+45/-1） |
| 触发条件 | `show=true && insight=null`（即首次点击 AI 卡到接口返回之间的窗口期） |
| 视觉 | 1 个 86% 宽度 headline skeleton + 3 个 chip skeleton（label / value / delta 三段），动画 pulse |
| refresh 行为 | 保留旧 insight + 按钮 `:loading` 态，**不**重新渲染 skeleton（避免闪烁） |
| 测试 | AiDrawer.spec.ts +3 用例（13 → 14）：skeleton 渲染 / 4 段结构 / 不渲染刷新按钮 |
| data-testid | `ai-drawer-skeleton` / `ai-drawer-skeleton-headline` / `ai-drawer-skeleton-chip` / `ai-drawer-skeleton-item` |

### 2.2 AI 角标 hover tooltip（v1.2.5 #2）

| 维度 | 详情 |
|------|------|
| 文件 | `frontend/src/views/HomeView.vue`（+23/-9） |
| 包装 | `ElTooltip` 包裹原有 `.home-view__source-badge` |
| placement / effect | top / light（浅色气泡，不挡卡） |
| 文案 | source=`llm` → "本次洞察由 AI 模型实时生成"；source=`template` → "模板生成的默认洞察（AI 模型暂不可用）" |
| a11y | 角标 `role="status"` + `aria-label` 保留原有；tooltip 通过 ElTooltip 自带的 popper aria 关联 |
| 测试 | HomeView.spec.ts +2 用例（12 → 14）：llm 文案断言 / template 文案断言（用 `findComponent(ElTooltip).props('content')` 验证，因 ElTooltip 不透传 data-testid 到 DOM） |

### 2.3 DailyView 错误态友好化（v1.2.5 #3）

| 维度 | 详情 |
|------|------|
| 文件 | `frontend/src/views/DailyView.vue`（+48/-0） |
| 触发条件 | `!loading && !daily && store.error`（即 daily API 失败后无数据） |
| 视觉 | ⚠️ emoji 图标 + ElEmpty `description="暂时无法获取日报数据，请稍后重试"` + ElButton type="primary" "重试" |
| 重试行为 | 复用 `onMounted` 同款调用：先 `fetchDaily(currentDate)`，若 URL 有 `?week=` 再 `fetchWeek(currentDate)` |
| 边界 | loading=true 时不渲染错误态（与 skeleton 互斥，避免视觉歧义）；`error=null && daily=null` 时仍走原有 `ElEmpty description="暂无日报数据，请稍后重试"`（理论上不触发，作为防御兜底） |
| 测试 | DailyView.spec.ts +3 用例（8 → 11）：错误态渲染 / 重试按钮触发 fetchDaily / loading 不渲染错误态 |
| data-testid | `daily-view-error` / `daily-view-error-description` / `daily-view-error-retry` |
