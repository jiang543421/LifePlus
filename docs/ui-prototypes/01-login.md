## 1. 故事 → 视图要素映射

| AC | UI 必须呈现 |
|---|---|
| AUTH-2 AC1：正确登录后跳「上次访问的页面或首页」 | 登录成功后跳 `?redirect=<path>` 或 `/` |
| AUTH-2 AC2：6 次失败限流，第 6 次明确文案 | 第 6 次起展示「请求过于频繁，请稍后再试」（错误码 1006） |

## 4. 字段清单（与后端 DTO 反查）

| 字段 | 必填 | 校验规则 | 后端 DTO |

## 5. 状态机

```
        ┌──────────┐
   →    │  IDLE    │ ← 进入页面 / 表单重置
        └────┬─────┘
             │ 点击「登录」
             ▼
        ┌──────────┐
        │ SUBMITTING│ ← 按钮 disabled + spinner，禁用密码框防止重复提交
        └────┬─────┘
             │ 后端响应
   ┌─────────┼─────────┐
   ▼         ▼         ▼
 code=0    code=1002  code=1006
 成功      凭证错     限流
   │         │         │
   │         ▼         ▼
   │     ElMessage    ElMessage
   │     "邮箱或       "请求过于
   │      密码错误"    频繁，请稍后再试"
   │         │         │
   │         └────┬────┘
   │              ▼
   │          ┌──────────┐
   │          │  IDLE    │（保留用户输入，不清空）
   │          └──────────┘
   ▼
 跳 ?redirect 或 /
```

### 5.1 1006 限流后的边界

- 1006 后**不立即禁用按钮**（限流是后端临时策略，1 分钟后自动放行）
- 用户看到限流提示后**可继续点击**——后端继续 1006，提示重复
- 不在前端做"软锁定"，避免与后端真实状态错位

## 6. 错误码 → 文案映射

| 后端 code | 前端文案 | 触发条件 |
|---|---|---|
| 1002 | 邮箱或密码错误 | 邮箱不存在 / 密码错（合并显示） |
| 1006 | 请求过于频繁，请稍后再试 | 1 分钟内第 6 次失败 |
| 0 | （无提示，直接跳转） | 登录成功 |
| 其它 | 系统繁忙，请稍后再试 | 网络/5xx（兜底） |

## 7. 与 1.4 落地的关联

- 用 `el-form` + `el-form-item` + `el-input` + `el-button` 组件
- Pinia `authStore.login(email, password)` action 调用 `api/auth.ts → POST /auth/login`
- `axios` 响应拦截器统一处理 1002 / 1006 → 本 view 仅消费 `authStore.error` getter
- 路由跳转：`route.query.redirect ?? '/'`

## 8. 不在本原型范围

- 第三方登录（PRD Out of Scope）
- 邮箱验证码找回密码（PRD Out of Scope，Phase 1.4 后评估）
- 滑动/图形验证码（MVP1 不画，避免引入额外依赖）
- 国际化文案（PRD Out of Scope）
