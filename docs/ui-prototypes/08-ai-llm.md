## 1. 故事 → UI 要素映射

| PRD 出处 | UI 必须呈现 |
|---|---|
| US-1 AC-1.1 | 智能卡 headline 显示 40-120 字中文，1-2 句，**至少包含 1 个判断** |
| US-1 AC-1.2 | 智能卡右上角 source 标签（"AI 生成"绿色 / "模板"灰色）|
| US-1 AC-1.3 | 智能卡 loading 时 headline 区显示 "分析中..."（≥ 3s）|
| US-2 AC-2.1 | 抽屉底部显示"查看完整分析 →"链接（**替换 v2.0 的"v2.1 将推出..."文案**）|
| US-2 AC-2.2 | 点击跳转 `/ai-analysis`，复用缓存（不再请求）|
| US-3 AC-3.1 | 切换 `lp.ai.llm.provider=ollama` 重启后，智能卡和独立分析页功能不变 |
| 加载态 | headline 区 + chip 区均有骨架屏 |
| 空数据态 | 主文 = `headline.empty`，4 chip 全 `—`，无 advice/highlight 段落 |
| 降级态 | source 标签显示"模板"（灰色），用户无感 |

```
┌─────────────────────────────────────────────────────────────────┐

│                                                                 │
│  ┌──────────────────┬──────────────────┬──────────────────┐     │
│  │ 任务完成          │ 本周消费          │ 今日日程          │     │
│  │ 80%              │ ¥420             │ 4 项              │     │
│  │ 较昨日 +5pp ↑    │ 较上周 -12% ↓    │ 与昨日持平 —     │     │
│  └──────────────────┴──────────────────┴──────────────────┘     │
│                                                                 │
│  生成于 08:30 · 缓存 1h23m 前                                    │
└─────────────────────────────────────────────────────────────────┘
```

**v2.1 改动（vs v2.0）**：

| 元素 | v2.0 | v2.1 |
|---|---|---|
| 标题栏按钮 | `[刷新][查看明细]` | `[AI 生成][刷新][ⓘ]` |
| 顶部标签 | ❌ 无 | ✅ `source` 标签（v2.1 新增）|
| headline | 1 句（任务 + 消费）| 2 句（判断 + 建议，可被 LLM 扩展）|
| 底部时间 | "生成于 08:30" | "生成于 08:30 · 缓存 1h23m 前"（含 freshnessSeconds 反算）|

**布局规则**：

- 桌面 `el-row :gutter="16"` 包裹 3 chip；`el-col :xs=24 :sm=8` 三等分
- 主文 `font-size: 18px; font-weight: 600; line-height: 1.5`
- chip 卡 `padding: 12px 16px; background: var(--el-fill-color-light)`
- source 标签：`<el-tag size="small" :type="source === 'llm' ? 'success' : 'info'" effect="plain">`
  - `source === 'llm'` → 绿色"AI 生成"
  - `source === 'template'` → 灰色"模板"
  - `source == null`（v2.0 旧缓存）→ 不显示标签
- 趋势箭头：UP = `↑` 绿色；DOWN = `↓` 红色；FLAT / NONE = `—` 灰色

**整卡可点击**：点击 headline / chip 区 → 触发抽屉（与 ⓘ 按钮等价）。

**布局规则**：

- 整页容器 `<el-main>` 或 `<div class="ai-analysis">`，`max-width: 960px; margin: 0 auto; padding: 24px`
- 顶部栏：`<el-page-header>` 或简化 `<div>`：返回箭头 + 标题 + 刷新按钮 + source 标签
- Headline 卡片：`<el-card shadow="never">` + `<h1>` 样式
- Advice 卡片：`<el-card shadow="never">` + 💡 icon + `<p>`
- Highlight 卡片：`<el-card shadow="never">` + ✨ icon + `<ul>` bullet 列表
- 关键指标：`<el-row :gutter="16">` + `<el-col :xs=12 :sm=6>` 4 等分
- 底部信息：`<p class="footnote">` 灰字

**source 标签逻辑**（与卡片、抽屉一致）：
- `source === 'llm'` → 绿色"AI 生成"
- `source === 'template'` → 灰色"模板"
- `source == null`（理论上不该出现）→ 不显示

**advice / highlight / mood / llmMeta 渲染规则**：
- LLM 成功 → 全部渲染
- 模板降级 → advice / highlight / mood / llmMeta 均为 `null` → **只渲染 headline 卡片 + 关键指标卡片**（advice/highlight 区块隐藏）
- 空数据 → headline = `headline.empty`，advice/highlight 区块隐藏，关键指标 4 chip 全 `—`

### 3.3 降级状态（LLM 失败 → 模板）

**用户感知差异**：

| 元素 | LLM 成功 | 模板降级 |
|---|---|---|
| 智能卡 source 标签 | 🟢 "AI 生成" | ⚪ "模板" |
| 智能卡 headline | 2 句（含判断 + 建议）| 1 句（任务 + 消费拼接）|
| 抽屉 source 标签 | 🟢 "AI 生成" | ⚪ "模板" |
| 抽屉内容 | headline + 3 指标 + 来源说明 | 同（headline 略短）|
| 独立分析页 Advice 区 | ✅ 显示 | ❌ **隐藏** |
| 独立分析页 Highlight 区 | ✅ 显示 | ❌ **隐藏** |
| 独立分析页 4 chip | ✅ 显示 | ✅ 显示 |
| 底部"v2.2 将加入..." | ✅ 显示 | ✅ 显示 |

> **关键 UX**：模板降级时用户感知差异最小化（source 标签 + 缺少 advice/highlight）；无错误弹窗、无 loading 延长。

## 4. 交互流程

### 4.1 状态机（智能卡 + 抽屉）

```

       mount ─────▶│   LOADING   │──── 200 ───▶ ┌─────────────┐
                   │ (skeleton)  │               │    READY    │
                   └──────┬──────┘               └──────┬──────┘

                          ▼                           │ click ⓘ / 整卡

                   │    ERROR    │            ┌─────────────┐
                   │ (fallback)  │            │  DRAWER     │
                   └──────┬──────┘            │  (open)     │
                          │ 自动重试 1 次     └──────┬──────┘
                          ▼                          │ click "查看完整分析 →"
                   ┌─────────────┐                  ▼
                   │    READY    │          ┌─────────────┐
                   └─────────────┘          │  ROUTE      │
                                             │ /ai-analysis│
                                             └─────────────┘
```

### 4.2 状态机（独立分析页）

```
                   ┌─────────────┐
       enter ─────▶│   LOADING   │──── 200 ───▶ ┌─────────────┐
                   │ (skeleton)  │              │    READY    │
                   └──────┬──────┘              └──────┬──────┘
                          │ 5xx / 1501                │
                          ▼                           │ click 刷新
                   ┌─────────────┐                    ▼
                   │    ERROR    │◀──────  失败 ── ┌─────────────┐
                   │ (fallback)  │                │  REFRESHING │
                   └──────┬──────┘                └─────────────┘
                          │
                          ▼
                       READY (auto retry 1x)
```

### 4.3 时序：用户进入首页 → 智能卡 → 抽屉 → 独立分析页

```
[1] 用户登录 → 进入 /home
        │
        ▼
[2] HomeView mounted
        │  ├─ 触发 AiInsightStore.load()
        ▼
[3] GET /api/v1/ai/insight/today  (Authorization: Bearer)
        │
        │  ├─ 后端：缓存命中 (lp:ai:insight:<userId>) → 直接返 (含 source)
        │  └─ 后端：缓存 MISS → 计算 → 写缓存 → 返
        ▼
[4] 响应到达 → 更新 aiInsight store
        │
        ▼
[5] 智能卡渲染 (READAY 态)
        │  ├─ headline + 3 chip + source 标签
        │  └─ [ⓘ] 按钮可点击
        ▼
[6] 用户点击 ⓘ / 整卡 → 抽屉打开
        │  ├─ AiDrawerVisible = true
        │  ├─ 复用 store.aiInsight (不重新请求)
        │  └─ 渲染 headline + 3 指标 + 来源说明 + 底部"查看完整分析 →"
        ▼
[7] 用户点击 "查看完整分析 →"
        │  ├─ router.push('/ai-analysis')
        │  └─ AiDrawerVisible = false (关闭抽屉)
        ▼
[8] AiAnalysisView mounted
        │  ├─ 检查 store.aiInsight 是否新鲜 (<6h)
        │  │  ├─ 是 → 直接用
        │  │  └─ 否 → 触发 GET /api/v1/ai/insight/analysis
        ▼
[9] 独立分析页渲染 (4 段内容)
        │  ├─ Headline 大字
        │  ├─ Advice 卡片 (LLM 才有)
        │  ├─ Highlight 卡片 (LLM 才有)
        │  ├─ 4 chip 关键指标
        │  └─ 底部 "v2.2 将加入..." 提示
        ▼
[10] 用户点击 ← 返回 → router.push('/home')
        │  └─ 智能卡仍可看到（缓存命中）
```

### 4.4 时序：用户点击智能卡刷新按钮

```
[1] 用户点击 [刷新]
        │
        ▼
[2] aiLoading = true (整卡 skeleton)
        │
        ▼
[3] POST /api/v1/ai/insight/refresh  (Authorization: Bearer)
        │  ├─ 后端：删缓存 + 重算
        │  │  ├─ 成功 → 200 + 新数据
        │  │  ├─ 1006 → 429 (限流，3/min/user)
        │  │  └─ 1501 → 503 (LLM + 模板双失败)
        ▼
[4] 响应处理
        │  ├─ 200 → 更新 aiInsight + 重渲染 (source 可能变化)
        │  ├─ 1006 → ElMessage.error('操作过于频繁，请稍后再试')
        │  └─ 5xx → ElMessage.error('数据异常，请稍后重试')
        ▼
[5] aiLoading = false
```

按钮在 `aiLoading=true` 时 `loading` 旋转 + disabled。

### 4.5 API 调用矩阵

| 用户动作 | 触发 API | 限流 | 错误处理 |
|---|---|---|---|
| 进入首页 | GET `/today` | 30/min/user (1006) | 5xx → fallback.headline + Toast |
| 智能卡刷新 | POST `/refresh` | **3/min/user** (1006) | 同上 |
| 点击 ⓘ 打开抽屉 | ❌ 无（复用 store）| — | — |
| 点击"查看完整分析 →" | router.push + GET `/analysis`（如缓存过期）| 30/min/user (1006) | 同上 |
| 独立分析页刷新 | POST `/refresh` | **3/min/user** (1006) | 同上 |
| 点击 ← 返回 | router.push('/home') | — | — |

> **`/analysis` 与 `/today` 共享缓存**（同一 Redis 键 `lp:ai:insight:<userId>`），独立分析页只在缓存过期时才发请求。

### 5.3 Pinia Store 复用

```typescript
// frontend/src/stores/aiInsight.ts（v2.0 沿用 + 扩展）
export const useAiInsightStore = defineStore('aiInsight', () => {
  const insight = ref<AiInsight | null>(null);
  const loading = ref(false);
  const error = ref<ApiError | null>(null);

  async function load() {
    loading.value = true;
    try {
      const r = await api.get('/api/v1/ai/insight/today');
      insight.value = r.data;
    } catch (e) {
      error.value = e as ApiError;
    } finally {
      loading.value = false;
    }
  }

  async function refresh() {
    loading.value = true;
    try {
      const r = await api.post('/api/v1/ai/insight/refresh');
      insight.value = r.data;
    } catch (e) {
      error.value = e as ApiError;
    } finally {
      loading.value = false;
    }
  }

  return { insight, loading, error, load, refresh };
});
```

> **AiDrawer 和 AiAnalysisView 共用同一 store**；抽屉打开不发起新请求；独立分析页通过 `insight.value` 直接读，避免重复请求。
