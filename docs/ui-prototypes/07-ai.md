## 3. 交互流程

### 3.1 点击卡片「查看明细」

```
[用户点击] Card footer [查看明细]
   │
   ▼
HomeView.aiDrawerVisible = true
   │
   ▼
<AiDrawer :payload="aiInsight" v-model:visible="aiDrawerVisible" />
   │
   ▼
抽屉从右滑入 280ms（el-drawer 默认）
   │
   ▼
渲染 5 行明细（复用 aiInsight.chips + Service 扩展字段）
```

注意：抽屉**不发起新请求**，复用 HomeView 已加载的 `aiInsight`。

### 3.2 点击卡片「刷新」

```
[用户点击] Card footer [刷新]
   │
   ▼
HomeView.aiLoading = true
   │
   ▼
GET /api/ai/insight  (Authorization: Bearer)
   │
   ├─ 200 + 数据 → 更新 aiInsight + 重新渲染 chip delta
   ├─ 1006 → ElMessage.error('操作过于频繁，请稍后再试')
   └─ 5xx → ElMessage.error('数据异常，请稍后重试')
   │
   ▼
HomeView.aiLoading = false
```

按钮在 `aiLoading=true` 时 `loading` 旋转 + disabled。

### 3.3 抽屉内关闭

| 触发 | 行为 |
|---|---|
| 右上 ✕ | `aiDrawerVisible = false` |
| Esc 键 | 同上（el-drawer 默认） |
| 遮罩点击 | 同上（`:close-on-click-modal="true"`） |
| 「关闭」按钮 | 同上，footer 右下 |

关闭动画 280ms 后组件 unmount。

## 4. 状态机

```
                ┌─────────────┐
   mount ─────▶ │   LOADING   │ ──── 200 ──▶ ┌─────────────┐
                └─────────────┘              │    READY    │
                       │                     └──────┬──────┘
                       │ 5xx / 1006                 │
                       ▼                            │
                ┌─────────────┐              click 刷新
                │    ERROR    │ ◀─────────────┘
                │ (Toast)     │  ────── 自动重试 1 次
                └──────┬──────┘              │
                       │                     ▼
                       └─────────────▶ READY
```

错误态不破坏卡片布局：chip 仍渲染 `—` 占位。

## 5. 与现有首页卡的关系

- 现有 6 卡（任务 / 日历 / 消费 / 饮食 / 设置 / 占位）保持不变
- AI 卡插入到第 1 位（最前），其下为任务卡（第 2 位）
- 命名：前端常量 `HOMEPAGE_CARDS` 在数组头部插入 `{ key: 'ai', title: '今日洞察', placeholder: false, component: AiInsightCard }`
- 移动端单列布局不变
