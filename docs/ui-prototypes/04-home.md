### 2.1 布局规则

- **网格**：桌面 3 列；平板 2 列；手机 1 列
- **问候语**：根据系统时间显示「早上好（5-11） / 中午好（12-13） / 下午好（14-18） / 晚上好（19-4）」
- **日期格式**：`YYYY-MM-DD 星期X`（zh-CN，使用 dayjs）

### 2.2 激活卡的内部结构

| 卡 | 上 | 中 | 下 |
|---|---|---|---|
| 今日待办 | 标题「今日待办」 | 大数字徽章 `5` | 列表 3 条（标题 + dueTime 简略） |
| 今日事件 | 标题「今日事件」 | 大数字徽章 `2` | 时间线 2 条（HH:mm + 标题） |

「查看全部 →」 router-link 跳 `/tasks` 与 `/plans`。

### 2.3 禁用卡的视觉

- 整卡灰度 + 半透明
- 中部居中显示「敬请期待」+ 灰色小字「v1.1 即将推出」
- 鼠标悬浮 cursor: not-allowed；点击无反应
- 不画「敬请期待」图标，避免又一轮图标设计

## 5. 状态机

```
        ┌──────────┐
   →    │ LOADING  │ ← 进入页面
        └────┬─────┘
             │ store.fetchHomeSummary() 完成
   ┌─────────┼──────────┐
   ▼         ▼          ▼
 空数据    有数据      接口失败
   │         │          │
   ▼         ▼          ▼
 卡片显示   卡片显示    ElMessage
 "暂无数据"  数字+列表   "首页加载失败，稍后重试"
                          │
                          ▼
                    ┌──────────┐
                    │  EMPTY   │ ← 可重试按钮
                    └──────────┘
```

### 5.1 失败可重试

失败态显示「重试」按钮 → 重新 `fetchHomeSummary`。**不画整页骨架屏**，局部红 banner + 重试按钮即可（MVP1 简单原则）。

## 6. 错误码 → 文案映射

| 触发 | 文案 | 行为 |
|---|---|---|
| 接口 401 + refresh 成功 | （无感知，继续显示） | 拦截器静默处理 |
| 接口 401 + refresh 失败 | （跳 `/login?redirect=/`） | 全局退出登录 |
| 接口 5xx / 网络断 | 首页加载失败，请稍后再试 | 显示重试按钮 |
| 数据为空 | 今日待办：暂无 / 今日事件：今日无安排 | 空态文案 |

## 7. 与 1.4 落地的关联

- 组件：`el-row` + `el-col`（`:xs=24 :sm=12 :md=8` 实现响应式 1/2/3 列）
- 卡片：`el-card` + `v-for` 渲染 6 个 `ModuleCard` 实例
- 数据：`homeStore.summary`（`tasksToday`, `plansToday`）由 `homeStore.fetchSummary()` 注入
- 1.4 时 `tasksToday` / `plansToday` 写死 0 / []（占位逻辑先行）
- 1.4 不引入 `homeStore`，直接在 `HomeView.vue` 内 `ref` 写死即可；Phase 4 再升级到 Pinia store

### 7.1 Phase 1.4 vs Phase 4 拆分

| Phase | 任务 |
|---|---|
| 1.4 | 6 张卡渲染 + 激活 2 张 + 数字 0 + 空态 + 重试机制 |
| 4 | 接后端真实接口 + Pinia `homeStore` 抽离 |

## 8. 不在本原型范围

- 任务/日程的内部 CRUD（MVP1 不做，分别 Phase 2/3）
- 数字拖动 / 卡片缩放（PRD 未提）
- 卡片自定义排序（PRD 未提）
- 卡片点击进入二级详情（仅「查看全部 →」跳列表页，不在本首页渲染）
- 跨卡片"添加任务/事件"快捷入口（保持首页只读，简化为聚合视图）
- 数据可视化图表（消费/AI 卡引入时再做）
