## 1. 模块概述

日程模块让用户在 LifePulse 内集中管理「什么时候要做什么」：

1. **月历总览**：一眼掌握本月安排密度。
2. **快速录入**：1 分钟内创建一条事件，仅填标题与起止时间即可。
3. **全天事件**：支持生日、节假日等不需要具体时刻的事件类型。
4. **承接任务**：作为任务「所属日程」的承载者，让任务能挂到日程下。

属于「期望需求」级别的核心模块——与任务模块并列承担用户日常回访动机。

## 2. 功能清单（MVP）

### 2.1 事件 CRUD

- [ ] 创建日程事件（标题、起止时间必填；地点、备注、提醒时长可选）
- [ ] 编辑日程事件
- [ ] 删除日程事件（软删除，30 天后清理）
- [ ] 起止时间校验：结束必须晚于开始
- [ ] 支持跨日事件（开始与结束不在同一天时仍正确显示）

### 2.2 视图与查询

- [ ] 日历视图（月视图，按日标记事件点）
- [ ] 进入日历页默认显示当月视图
- [ ] 有事件的日期用色点标记，悬停可看到当天事件数
- [ ] 点击某一天，右侧/下方展开当天事件列表
- [ ] 日历按月范围查询本月所有事件
- [ ] 日程列表分页（每页 20 条）
- [ ] 日程详情查看
- [ ] 提醒时长字段占位（仅记录，不发推送通知；Phase 2 再接入推送）

### 2.3 全天事件

- [ ] 创建事件时勾选「全天」，则起止时间字段被隐藏，仅记录日期
- [ ] 全天事件在日历上以顶部色条标识，与时段事件视觉区分

### 2.4 弹窗与详情

- [ ] 在日历上点击某一天，弹出新建事件弹窗
- [ ] 弹窗内可切换为「编辑事件」模式
- [ ] 日程详情页可看到「本日程下的任务」区域（联动任务模块）

### 2.5 首页集成

- [ ] 首页计划卡显示今日事件数
- [ ] 点击首页计划卡跳转到日历视图

## 3. 用户故事

### 故事 PLAN-1：月历总览
> 作为 **用户**，我需要 **在月历上看到本月所有事件标记**，以便于 **一眼掌握这周的安排密度**。

**验收标准（AC）**：
1. 进入日历页默认显示当月视图。
2. 有事件的日期用色点标记，悬停可看到当天事件数。
3. 点击某一天，右侧/下方展开当天事件列表。

### 故事 PLAN-2：创建与编辑事件
> 作为 **用户**，我需要 **快速录入一条日程事件**，以便于 **不遗漏任何约定**。

**验收标准（AC）**：
1. 在日历上点击某一天，弹出新建事件弹窗，仅需标题与起止时间即可保存。
2. 起止时间校验：结束必须晚于开始。
3. 支持跨日事件（开始与结束不在同一天时仍正确显示）。

### 故事 PLAN-3：全天事件
> 作为 **用户**，我需要 **标记「全天」事件（如生日、节假日）**，以便于 **不需要给出具体起止时刻**。

**验收标准（AC）**：
1. 创建事件时勾选「全天」，则起止时间字段被隐藏，仅记录日期。
2. 全天事件在日历上以顶部色条标识，与时段事件视觉区分。

| 指标 | 目标 |
|---|---|
| 日历按月查询接口 P95 | ≤ 100 毫秒 |

## 5. 边界

**In Scope**：事件 CRUD / 软删除 / 月历视图 / 按月范围查询 / 分页 / 详情 / 新建编辑弹窗 / 全天事件 / 提醒时长占位字段 / 与任务的关联。

**Out of Scope**：
- 推送通知 / 邮件提醒 —— MVP1 仅记录字段。
- 重复事件（每周 / 每月循环） —— Phase 2 评估。
- 邀请他人参与事件 —— 团队功能，超出 MVP。
- 与外部日历（Google / 苹果）同步 —— Phase 2。
- 多时区 —— MVP1 仅按用户本地时区展示。
