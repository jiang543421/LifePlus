## 1. 总体愿景

Phase 2 把 LifePulse 从「任务 + 日程」扩展为「数字生活全维度助手」，覆盖四类延伸场景：**每日回顾、钱财流动、饮食摄入、智能化建议**。这四个模块统一遵守三条原则：

1. **数据驱动**：必须有 MVP1 已积累的任务/日程/消费数据作为输入才有意义。
2. **渐进式**：每个模块独立发布，不互相阻塞。
3. **可关闭**：用户可单独隐藏某个卡片，不被强推。

四个模块的产品愿景描述见下节。

## 2. 四大模块愿景

### 2.1 日报
**一句话定位**：每天 / 每周帮用户做一次「回顾与统计」，回答「这周 / 今天我做了什么、效率如何」。

**核心场景**：
- 晚 10 点提示「今日有 X 项待办完成 Y 项」自动回顾
- 周日晚提示「本周任务完成率 N%，建议下周…」式总结
- 每日一条记录（用户手写或自动从任务/日程聚合）

**与 MVP1 的关系**：依赖任务模块（完成事件）+ 日程模块（事件密度）。

---

### 2.2 消费
**一句话定位**：日常开支记账与统计，覆盖餐饮、购物、交通、订阅等大类。

**核心场景**：
- 一笔消费 30 秒内录入（金额 + 分类 + 备注）
- 月底自动生成「分类占比」饼图与「同比/环比」对比
- 跨年查询：年支出趋势 + 分类热力图

**与 MVP1 的关系**：独立模块，但日报模块可读取消费数据生成「本周消费概览」。

---

### 2.3 饮食
**一句话定位**：每日饮食记录 + 营养摄入估算，帮助用户建立均衡饮食习惯。

**核心场景**：
- 三餐 + 零食随手记录（含食物搜索与常用快捷项）
- 每日摄入热量 / 蛋白质 / 碳水 / 脂肪自动汇总
- 周对比：连续记录 7 天后给出「饮食均衡性」反馈

**与 MVP1 的关系**：独立模块；与消费模块可在「餐饮分类」上做数据互通（同一笔消费可同时记入饮食记录）。

---

### 2.4 AI 分析
**一句话定位**：基于已有任务 / 日程 / 消费 / 饮食数据，为用户生成趋势洞察与个性化建议。

**核心场景**：
- 「本月你最高效的工作日是周二/周三，建议把重要任务排到这两天的上午」
- 「你的咖啡消费在连续加班周显著上升，建议关注压力管理」
- 「本月运动相关任务完成率 30%，低于其他类别，要不要调整目标？」

**与 MVP1 的关系**：必须依赖 Phase 2 三个模块都已积累 ≥ 30 天数据后才能输出有意义的洞察——**因此 AI 分析是 Phase 2 的最后一个模块**，不在前三模块完成前启动。

---

## 3. 首页占位

在 MVP1 的首页上，日报 / 消费 / 饮食 / AI 分析四张卡片显示为「占位卡」，点击提示「即将上线」。

- 视觉与功能卡一致（白底圆角、淡蓝阴影）
- 点击后弹出轻提示（toast 形式）
- 不消耗数据接口调用

---

## 4. 启动优先级建议

| 模块 | 启动时机 | 依赖 |
|---|---|---|
| 日报 | Phase 2.1（最早） | 任务 + 日程（MVP1） |
| 消费 | Phase 2.2 | 无 |
| 饮食 | Phase 2.3 | 无（与消费可数据互通） |
| AI 分析 | Phase 2.4（最晚） | 消费 + 饮食 ≥30 天数据 |

> **关键决策**：AI 分析作为压轴模块，仅在前三个 Phase 2 模块已有数据沉淀后才启动，避免空泛结论。

---

## 5. 暂不规划（Not Planned）

- 社交分享 / 朋友圈打卡 —— 个人工具不需要
- 智能硬件接入（手环 / 体重秤） —— Phase 3+
- 多端同步（iCloud / Google Drive 备份） —— 隐私敏感，评估中
