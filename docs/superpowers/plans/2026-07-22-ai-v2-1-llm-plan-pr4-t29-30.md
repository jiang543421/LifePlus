### Task 29: pre-commit + gitleaks 防 API key 泄露

**Files:**
- Create: `.gitleaks.toml`（规则配置）
- Modify: `.github/workflows/ci.yml`（添加 gitleaks 步骤）
- Create: `.git/hooks/pre-commit`（手动安装）

> **硬约束标注**：CLAUDE.md §7.1 禁硬编码密钥 → 通过 pre-commit + gitleaks 双层防护。`.env` 已 `.gitignore`（v1.x 已加），本 task 加主动扫描。

- [ ] **Step 1: Create `.gitleaks.toml`**

```toml
title = "LifePulse gitleaks"
[extend]
useDefault = true

[[rules]]
id = "deepseek-api-key"
description = "DeepSeek API key pattern"
regex = '''sk-[a-f0-9]{32,}'''
tags = ["secret", "llm"]

[[rules]]
id = "lp-jwt-secret"
description = "LP_JWT_SECRET in source code"
regex = '''LP_JWT_SECRET\s*=\s*["'][^"']{16,}["']'''
tags = ["secret", "jwt"]
```

- [ ] **Step 2: 写 pre-commit hook**

`.git/hooks/pre-commit`:

```bash
#!/usr/bin/env bash
set -e
which gitleaks >/dev/null 2>&1 || { echo "gitleaks not installed, skipping"; exit 0; }
gitleaks protect --staged --redact --config .gitleaks.toml
```

```bash
chmod +x .git/hooks/pre-commit
```

- [ ] **Step 3: 在 CI 添加 gitleaks 步骤**

`.github/workflows/ci.yml` 追加：

```yaml
  gitleaks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          GITLEAKS_CONFIG: .gitleaks.toml
```

> **红线标注**：本 task 修改 CI 配置文件属敏感操作。**已确认属于"安全强化"，非密钥改动**，CLAUDE.md 红线指"修改密钥/token"，CI 规则文件不属于。

- [ ] **Step 4: 验证 hook（可选，需本地装 gitleaks）**

```bash

git add /tmp/test-secret.txt

```

- [ ] **Step 5: Commit**

