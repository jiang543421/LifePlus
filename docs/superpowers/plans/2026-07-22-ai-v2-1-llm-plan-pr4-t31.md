```markdown
# v2.1.0-ai — 2026-07-22

## Highlights

- AI 洞察接入 LLM（DeepSeek V3 为主，Ollama 为备），三层降级链路
- 独立分析页面 `/ai-analysis`
- 缓存 TTL 30min → 6h；refresh 限流 6/min → 3/min
- 日配额 50/user/day；熔断器自动保护
- 5 个新错误码（1510/1511/1512/1513）

## 配置

新增环境变量：

\`\`\`bash
LP_LLM_PROVIDER=deepseek    # 或 ollama
LP_LLM_API_KEY=sk-xxx       # 仅 deepseek 模式
LP_LLM_BASE_URL=https://api.deepseek.com
LP_LLM_MODEL=deepseek-chat
LP_LLM_DAILY_ENABLED=true
LP_LLM_DAILY_QUOTA=50
LP_LLM_PROMPT_MAX_TOKENS=1500
LP_LLM_RESPONSE_MAX_TOKENS=300
LP_LLM_TIMEOUT_MS=8000
LP_LLM_RETRY=0
\`\`\`

## 测试

- 后端单测：14 个新增 case（LLM 客户端 / prompt builder / parser / quota / circuit / generator）
- 后端 IT：6 个 Testcontainers case
- 前端单测：5 个 case（store + 3 components + 1 guard）
- E2E：2 个 Playwright spec（5 用例）
- 覆盖率：后端 Service ≥ 80%（JaCoCo gate），前端 store 100%

## 升级步骤

1. 拉取 `v2.1.0-ai` tag
2. 在 `.env` 配置 `LP_LLM_API_KEY`（从 DeepSeek 控制台获取）
3. `docker compose up -d --build`
4. 不需要 DB migration（v2.1 无新表）
5. 默认 `LP_LLM_DAILY_ENABLED=true`；不需要 LLM 的用户可在 `.env` 设为 `false`

## 回滚

\`\`\`bash
git revert <PR1-commit> <PR2-commit> <PR3-commit> <PR4-commit>
# 或：
git checkout v2.0.0-ai
\`\`\`

## 已知限制

- LLM 输出受 prompt 影响，可能偶发不准确；UI 已展示 source 标签让用户识别
- DeepSeek 限流（默认 60 RPM）；超额触发熔断器（5 分钟自动恢复）
- Ollama 模式无 API key，但需要本地 GPU；不推荐生产
```
