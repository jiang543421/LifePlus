# LifePulse AI 分析 v2.0 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 LifePulse 首页把"AI 分析"占位卡升级为智能洞察卡（headline + 3 chip + 详情抽屉），基于 5 类数据自动生成中文洞察，纯规则 + 模板，Redis 30 分钟缓存。

**Architecture:** 沿用日报 `MetricProvider` 模式 → 5 个 `AiInsightProvider` 串行采集 → `AiTemplateEngine` 用 JDK `String.format` 渲染 headline → `AiInsightService` 编排缓存（命中不延长 TTL）→ `AiInsightController` 暴露 2 个端点。降级 3 层：模块未上线（`isEnabled=false`）/ 单 provider 异常（catch+skip）/ 整体不可用（1501）。前端 `aiInsight` Pinia store + `AiInsightCard` / `AiChipItem` / `AiInsightDrawer` 组件，替换 `HomeView` 占位卡。

**Tech Stack:**
- 后端：Spring Boot 3.3.5、MyBatis-Plus 3.5.6、Spring Data Redis、Spring Security 6、JUnit 5、Mockito、AssertJ、Testcontainers（mysql:8、redis:7）
- 前端：Vue 3.4 + TypeScript 5 strict + Pinia 2 + Axios 1.7 + Element Plus 2.8 + Vitest 2 + @vue/test-utils 2.4 + Playwright

## Global Constraints

照搬 `CLAUDE.md` §1-9 的硬约束——所有任务默认遵守：

- **JDK 17**（不可擅自升 21，须 issue + 迁移说明）；Spring Boot 3.3.5；Maven 3.9+
- **后端** 4 空格缩进 / **前端** 2 空格缩进；双引号；分号；LF 行尾；文件尾必换行
- 函数 ≤ 50 行；文件 ≤ 800 行；嵌套 ≤ 4 层；魔法数字必须命名常量
- **不可变性**：Java `record` / Lombok `@Builder` / 前端 `{...obj, field: value}`；集合 `List.copyOf(...)`
- **错误处理**：业务异常 `BusinessException(code, msg)`，禁 `Optional.empty()` 隐式掩盖越权
- **输入验证**：所有 DTO `@Valid`；`@NotBlank`/`@Email`/`@Size`/`@Pattern`
- **安全**：禁硬编码密钥；JWT HS256 ≥ 32 字节；refresh 旋转；所有 `{id}` 端点先 `UserContext.current()` 过滤越权
- **测试**：Service 行覆盖 ≥ 85%；Controller 鉴权/错误码 100%；Provider 各实现 ≥ 80%
- **跨用户越权**：禁字符串拼 SQL；`@WebMvcTest` 100% 鉴权覆盖
- **日志**：禁打 password/token/email 全；仅打 `userId` 与 `email` 前 2 位
- **Git**：分支 `feat/ai-v2.0`；commit `<type>(<scope>): <subject>`，英文，首字母小写，≤ 72 字符，无句号
- **不引入 LLM**、**不引入新表**、**不引入新第三方依赖**
