# Implementation Plan: LifePulse v1.2.3 — Daily Report 模块

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> 本计划仅依赖 `docs/prd/05-daily-report.md`(PRD v1.0 草案,6 节设计 + 8 项决策已确认)+ `docs/specs/08-daily-report-design.md`(落档中)。
> 一切以 `CLAUDE.md` + 全局规则为准;Java 细则遵循 `~/.claude/rules/ecc/java/*`。
> 与 settings v1.1 + expense v1.2.1 + diet v1.2.2 已落地能力(`UserContext` / `JwtAuthFilter` / `ErrorCode` / `RateLimiter` / `t_task` / `t_plan` / `t_expense`)完全复用,**不引入新基础设施、不新建业务表**。

---

## 1. Goal Recap

- **v1.2.3 ships**:日报模块端到端闭环 —— 用户在 `/daily` 一个视图里看到「今天/本周做了什么、效率如何、钱花在哪」,**请求时实时聚合 `t_task` / `t_plan` / `t_expense` 三表,只读不可改,零录入成本**;激活首页占位卡为模块卡。
- **defer to v1.3+ / v2.x**:月报视图、AI 摘要、报告导出、自定义指标、趋势图标、时区设置、报告对比、报告订阅、饮食模块正式接入(E1-E10)。

---

## 2. Acceptance Criteria

> 编号与 PRD §1.2 SMART 指标对齐;**不可让步**。

| 编号 | 验收点 | 测试要求 |
|---|---|---|
| AC-1 Migration | V5 仅加索引,**不新建业务表**;`t_task.idx_user_completed_at`、`t_plan.idx_user_start_at`、`t_expense.idx_user_occurred_at` 全部存在 | `mvn flyway:info` + IT 启动自动 migrate |
| AC-2 Provider 接口 | `MetricProvider<T>` 接口 + 4 实现:Task / Plan / Expense / Diet | `MetricProviderTest` ≥ 4 case |
| AC-3 Diet 锁定 | `DietMetricProvider.aggregate()` **永远**返回 `enabled=false, value=null`,单测 + 集成测试双重断言 | `DietMetricProviderTest` 永久锁死 |
| AC-4 Service | `DailyReportService.aggregateDaily(userId, date)` + `aggregateWeekly(userId, isoWeek)`;沿用 `UserContext.current()`;输入校验(1004) | `DailyReportServiceTest`(Mockito, ≥6 case) |
| AC-5 Controller | 2 端点 `GET /api/daily?date=` / `GET /api/daily/week?week=`;鉴权 / 校验 / 未来日 1004 路径 100% 覆盖 | `DailyReportControllerWebTest`(@WebMvcTest, ≥7 case) |
| AC-6 Frontend | 5 interface + API + Pinia store + 4 Card(任务/日程/消费/饮食) + WeeklyComparisonBar + DailyReportView + 路由 `/daily` + 首页卡激活 | store ≥ 9 case + 关键组件 100% |
| AC-7 E2E | 主页跳转 / 正常日 / 空日 / 饮食 disabled / 未来日 1004 / 周报对比 / 跨用户隔离 / 上下日导航 8 用例 | `e2e/daily/daily-report.spec.ts`(Playwright) |
| AC-8 性能 | 日报 P95 ≤ 200ms,周报 P95 ≤ 300ms(本地,1w 行种子) | IT 内 1w 行种子 + 计时断言 |

**覆盖率硬指标**:Service 行覆盖 ≥80%(JaCoCo 闸门);前端 store 行覆盖 ≥70%;关键组件(DailyReportView + 4 Card + WeeklyComparisonBar)100%。
