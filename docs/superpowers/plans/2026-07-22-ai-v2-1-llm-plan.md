# AI 分析模块 v2.1 (LLM 增强) — 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 v2.0 智能卡 + 抽屉基础上叠加 LLM 洞察升级，并兑现 v2.0 留的 `/ai-analysis` 独立分析页承诺；通过 3 层降级（L1 LLM → L2 模板 → L3 1501）保证 v2.0 模板引擎仍为最终回退。

**Architecture:**
- 后端：v2.0 Service 注入 `LlmInsightGenerator`（手写 `RestClient` 调 OpenAI 兼容协议），Provider 聚合层与缓存层零改动。
- Redis 命名空间扩展：洞察缓存 TTL 30min→6h；新增 `lp:ai:quota:*` 配额键 + `lp:ai:circuit:*` 熔断键。
- 前端：智能卡加 `source` 标签；抽屉底部加"查看完整分析 →"路由；新增 `AiAnalysisView.vue`。
- 零 DB schema 变更、零 Maven 依赖变更、零 Spring Bean 冲突。

**Tech Stack:**
- 后端：Spring Boot 3.3.5 + Spring `RestClient`（Boot 内置）+ Spring Data Redis + JUnit 5 + Mockito + Testcontainers
- 前端：Vue 3.4 + TypeScript 5 + Element Plus 2.8 + Pinia 2 + Axios 1.7 + Vitest + Playwright
- LLM：DeepSeek V3 (`deepseek-chat`) 主选 + Ollama (`deepseek-r1:8b`) 备选（配置切换）

## Global Constraints

这些约束来自 `CLAUDE.md` §11（AI 模块规范）和 §1-9（项目级规范）。**每个任务的实现必须隐式遵守**：

- **OpenJDK 17 + Maven 3.9+ + Spring Boot 3.3.5**（CLAUDE.md §2.1）
- **零新增 Maven 依赖**（CLAUDE.md §11.1 #3）：手写 `RestClient` 调 OpenAI 兼容协议，不引入 Spring AI
- **零新增数据库表、零 Flyway 迁移**（CLAUDE.md §11.1 #1, #2）
- **2 空格缩进（YAML/TS/Vue），4 空格（Java/SQL）**（CLAUDE.md §4.2）
- **不可变优先**：record + final + `@JsonInclude(NON_NULL)`（CLAUDE.md §4.1 + §11.1）
- **跨用户隔离**：所有 Redis 键按 `<userId>` 隔离；端点不接受 userId 参数（CLAUDE.md §7.2 + §11.1 #4）
- **依赖方向单向**：`Web → Service → LlmInsightGenerator → LlmClient`（CLAUDE.md §11.1 #5）
- **`LP_LLM_API_KEY` 占位符**：deepseek 模式必填，启动期 fail fast（CLAUDE.md §11.2）
- **3 层降级**：L1 失败不重试；L2 失败抛 1501；用户感知最小化（CLAUDE.md §11.3）
- **Redis 命名空间**：`lp:ai:insight:<userId>` / `lp:rl:ai:*` / `lp:ai:quota:*` / `lp:ai:circuit:*`（CLAUDE.md §11.4）
- **错误码新增**：1510/1511/1512/1513（CLAUDE.md §11.6，全部 Service 内部 catch 走 L2）
- **测试覆盖率硬阈值**：Generator ≥ 90% / Service ≥ 85% / Client ≥ 80% / IT ≥ 6 用例（CLAUDE.md §11.7）
- **commit message 格式**：`<type>(<scope>): <subject>`，subject 英文、首字母小写、≤ 72 字符、无句号（CLAUDE.md §5.2）
- **TDD 强制**：每个功能改动按 Red → Green → Refactor（CLAUDE.md §6.2）
- **Java 测试命名**：`methodName_stateUnderTest_expectedBehavior`（CLAUDE.md §6.3）
- **TS 测试命名**：`describe('Component') + it('behavior')`（CLAUDE.md §6.3）
- **E2E 命名**：`<feature>.spec.ts`（CLAUDE.md §6.3）
