- [ ] **Step 3**: 创建 `MetricProvider<T>` 接口

```java
package com.lifepulse.daily.provider;

import java.time.LocalDate;

public interface MetricProvider<T> {
  /** 聚合单日指标;LocalDate 由 Service 转 Asia/Shanghai 边界 */
  T aggregateDaily(long userId, LocalDate date);

  /** Provider 元数据名称(用于日志) */
  default String name() {
    return getClass().getSimpleName();
  }
}
```

> **设计决策**:`Provider<T>` 不强制 `aggregateWeekly`,因为周报聚合由 `DailyReportService` 逐日循环 Provider 调用,Provider 保持单日接口简单。
