# [Phase 2+] 设置页 v1.1（改昵称 / 改密码 / 注销账号）

**描述**：`/settings` 当前仅占位 v1.1。MVP2 第一阶段先把账号基础设置落地，三个动作：改昵称、改密码（需旧密码校验）、注销账号（需输入 + 二次确认 + 软删除）。

**Acceptance Criteria**：
- [ ] 后端新增 `PATCH /api/v1/users/me`（昵称）、`POST /api/v1/users/me/password`（旧+新）、`DELETE /api/v1/users/me`（注销）
- [ ] 三接口必须按 `UserContext.current()` 过滤；注销走逻辑删除（`@TableLogic`），所有 refresh token 一并 `revoked_at = NOW()`
- [ ] 改密码强制 revoke 所有现存 refresh token，下次需重新登录
- [ ] 前端 `SettingsView` 拆 3 个区块：资料 / 安全 / 危险区；`Element Plus 表单` + `FormRule` + `ElMessageBox` 确认
- [ ] Auth store 暴露 `updateProfile / changePassword / deleteAccount` 三个 action
- [ ] 单测：3 个 service 方法 + 3 个 store action + SettingsView 3 段；E2E：改昵称 → 重登能看到新昵称 / 改密码 → 旧 token 失效 / 注销 → 跳 /login 且 GET /users/me 返回 1001
- [ ] 跨用户越权路径必须拦截（用 Bob 的 token 改 Alice 的 userId 自动 1003）
